## 1. 프로젝트 목적

창업 지원사업에서 **멘토와 멘티를 연결하고, 멘토링 세션의 전 과정을
한곳에서 관리**하기 위해 만든 운영 도구입니다.

기존에는 멘토링 일정 조율, 일지 수합, 실적 집계를 **엑셀·이메일·메신저로
분산 처리**했습니다. 멘토링 허브는 이 흐름을 하나의 웹앱으로 통합하여
**담당자의 운영 부담을 줄이고, 멘토·멘티의 참여 경험을 단순화**하는 것을
목표로 합니다.

설계 철학은 **"가볍게, 그러나 제대로"** 입니다.
- 빌드 도구·프레임워크 없이 **단일 HTML 파일**로 배포 가능
- 백엔드는 **Supabase**(PostgreSQL + Auth + Realtime + Storage)로 위임
- 별도 서버 운영 없이 **정적 호스팅**만으로 동작

## 2. 해결하려는 실제 문제

실제 멘토링 사업 운영에서 반복적으로 발생하던 문제들을 해결합니다.

| 기존 문제 | 멘토링 허브의 해결 |
|-----------|------------------|
| 📅 **일정 조율 혼선** — 멘토 가능 시간과 멘티 희망 시간을 메신저로 주고받으며 누락·중복 발생 | 멘토가 가용 슬롯을 등록하면 멘티가 직접 예약, **동시성 안전(`FOR UPDATE`) 예약**으로 중복 차단 |
| 📝 **일지 수합의 번거로움** — 멘토가 양식에 맞춰 작성·제출하고 담당자가 취합 | 표준 일지 양식 + 사진 첨부 + 임시저장, **제출/미제출 자동 집계** |
| 📊 **실적 집계 수작업** — 월별·멘토별 세션 수를 엑셀로 수동 정리 | 기간별·멘토별 실적을 **대시보드에서 자동 산출** |
| 🔗 **매칭 과정의 불투명성** — 누가 누구와 매칭됐는지 추적 어려움 | 신청 기반 / 직접 매칭 **2가지 플로우 + 상태 추적** |
| 📬 **알림 사각지대** — 변경 사항을 당사자가 늦게 인지 | **Supabase Realtime**으로 상태 변경 즉시 반영 |

## 3. 주요 기능

### 🎓 멘토
- 가용 일정(슬롯) 등록·삭제, 미니 달력으로 현황 확인
- 예약 신청 수락/거절, 세션 완료 처리
- 멘토링 일지 작성(유형·소요시간·상담내용·사진 첨부) · 임시저장 · 수정
- 공지 열람, 담당자에게 요청/문의 제출

### 🚀 멘티
- 멘토 목록 조회 및 가용 시간 예약
- 매칭 신청(희망 분야·일정·요청사항)
- 내 멘토링 내역 확인, 세션별 피드백 작성
- 공지 열람, 요청/문의 제출

### 📊 관리자
- **전체 현황 대시보드** — 등록 인원·세션·일지 실적 요약
- **매칭 관리** — 신청 기반 매칭 + 직접 매칭(3탭 구성)
- 멘토/멘티 등록·관리, 일지 검색·열람
- 공지 작성, 요청 회신, 메시지 발송
- 역할 전환 미리보기(읽기 전용)

### 🔐 공통 기반
- Google 로그인 기반 인증(provider-agnostic 백엔드)
- 역할별 데이터 접근 제어(**RLS 정책** + RPC)
- 낙관적 UI 업데이트 + 실패 시 롤백
- XSS 방어(전 출력 이스케이프), 안전한 URL 처리

## 4. 향후 오픈소스 확장 계획

멘토링 허브는 현재 **특정 사업단 운영용**으로 개발되었으나,
**여러 기관이 재사용할 수 있는 오픈소스 멘토링 플랫폼**으로
확장하는 것을 지향합니다.

### 단기 (운영 고도화)
- [ ] 검색·페이지네이션(데이터 증가 대비)
- [ ] 멘토/멘티 본인 프로필 수정
- [ ] 일지·피드백 수정/삭제, 일정 변경
- [ ] 다중 관리자 지원 UI

### 중기 (재사용성)
- [ ] **설정 기반 커스터마이징** — 기관명·로고·테마 컬러·일지 양식을 코드 수정 없이 변경
- [ ] **다양한 로그인 수단** — 이메일/비밀번호, 매직 링크, 카카오 등 (백엔드는 이미 provider-agnostic)
- [ ] 엑셀 내보내기 고도화, 통계 API
- [ ] i18n(다국어) 지원

### 장기 (오픈소스화)
- [ ] **원클릭 배포 템플릿** — Supabase 스키마/RLS/RPC를 마이그레이션 1벌로 제공
- [ ] 설치 가이드·기여 가이드(CONTRIBUTING) 문서화
- [ ] 라이선스 확정 및 데모 인스턴스 제공
- [ ] 플러그인 구조 — 사업 유형별 모듈(창업/취업/연구 멘토링 등) 확장

> 💡 **기여 환영** — 기관 운영 경험에서 나온 실제 문제를 풀고 있어,
> 비슷한 고민을 가진 분들의 피드백과 기여를 기다립니다.
