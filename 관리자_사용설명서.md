# 멘토링허브 관리자 사용설명서

---

## 1. 로그인

1. 멘토링허브 접속
2. **Google로 로그인** 버튼 클릭
3. 관리자 계정으로 등록된 Google 계정 선택
4. 자동으로 관리자 대시보드 진입

> 관리자 계정은 Supabase users 테이블에 role='admin'으로 등록되어 있어야 합니다.

---

## 2. 사이드바 메뉴 구성

| 메뉴 | 설명 |
|------|------|
| 전체 현황 | 대시보드 통계 및 세션 관리 |
| 매칭 관리 | 멘토-멘티 매칭 처리 |
| 멘토 관리 | 멘토 목록, 알림 이메일 관리 |
| 멘티 관리 | 멘티 목록, 삭제 처리 |
| 일지 목록 | 멘토링 일지 열람 |
| 요청 관리 | 멘토/멘티 요청 회신 |
| 공지 작성 | 공지사항 등록/삭제 |
| Supabase 연동 | 시스템 설정 및 이메일 큐 |

---

## 3. 전체 현황 (대시보드)

### 3-1. 요약 통계 카드

화면 상단에 8개 핵심 지표가 표시됩니다.

- 등록 멘토 / 등록 멘티 / 가용 슬롯 / 신청 대기
- 확정 세션 / 완료 세션 / 제출된 일지 / 일지 미제출

### 3-2. 기간별 통계

탭을 선택하여 기간별 멘토링 현황을 조회합니다.

- **이번 주** : 이번 주 월~일 세션 통계
- **이번 달** : 이번 달 세션 통계
- **전체 누적** : 전체 기간 통계
- **커스텀** : 연/월 또는 연/월/주 단위로 직접 선택

각 탭에서 확인 가능한 정보:
- 총 세션 수, 완료/확정/대기/일지 건수
- 멘토별 실적표 (세션 수, 일지 수)

### 3-3. 일지 미제출 목록

완료되었으나 일지가 미제출된 세션 목록을 표시합니다.
멘토에게 제출을 요청할 때 참고합니다.

### 3-4. 멘토-멘티 매칭 현황표

모든 멘토-멘티 조합의 세션 횟수를 표로 보여줍니다.
- 총 횟수 / 완료 / 확정(예정) / 대기

### 3-5. 세션 목록

필터를 이용하여 세션을 조회합니다.

- **기간 필터** : 전체 / 이번 주 / 이번 달
- **상태 필터** : 전체 / 대기 / 확정 / 완료

각 세션에서 가능한 작업:
- 일지 보기 (제출된 경우)
- **세션 취소** : 대기/확정 상태의 세션을 관리자가 강제 취소

> 관리자가 세션을 취소하면 멘토와 멘티 양쪽에 이메일 알림이 발송됩니다.

---

## 4. 매칭 관리

매칭 관리는 3개 탭으로 구성됩니다.

### 4-1. 신청 탭

멘티가 신청한 매칭 요청을 처리하는 화면입니다.

**처리 흐름:**

```
멘티 매칭 신청 → [대기중] → 관리자가 멘토 선택 후 제안 → [제안됨]
→ 멘토가 수락 → [확정]  ← 멘토-멘티 모두 이메일 알림
→ 멘토가 거절 → [대기중] ← 관리자에게 이메일 알림 (재매칭 필요)
```

**작업 방법:**
1. 대기 중인 신청 목록에서 멘티의 희망 분야/일정/요청사항 확인
2. **멘토 선택 후 제안** 버튼 클릭
3. 적합한 멘토 선택 + 메모 입력 (선택)
4. **매칭 제안** 확인 → 멘토에게 이메일 발송

### 4-2. 제안 현황 탭

모든 매칭 제안의 진행 상태를 확인합니다.

- **상태 필터** : 전체 / 제안중 / 확정 / 거절 / 해제
- 확정된 매칭의 멘토링 활동 현황 (완료 건수, 예정 건수)
- **매칭 해제** : 확정된 매칭을 종료 처리

> 매칭 해제 시 해당 신청 건은 '종료' 상태로 완전히 마감됩니다. 재매칭 목록에 남지 않습니다.

### 4-3. 직접 매칭 탭

신청 없이 관리자가 직접 멘토-멘티를 매칭합니다.

1. 멘토 선택 (드롭다운)
2. 멘티 선택 (드롭다운)
3. 메모 입력 (선택)
4. **매칭 확정** 클릭 → 즉시 확정 (멘토 수락 과정 생략)

> 직접 매칭 시 멘토와 멘티 양쪽에 확정 이메일이 발송됩니다.

## 5. 멘토 관리

### 5-1. 멘토 목록 테이블

| 열 | 내용 |
|----|------|
| 이름/이메일 | 멘토 기본 정보 |
| 분야 | 전문 분야 |
| 소속 | 소속 기관 |
| 알림 이메일 | 매칭/세션 알림을 받을 별도 이메일 |
| 세션 수 | 총 세션 횟수 |
| 완료 | 완료된 세션 수 |
| 일지율 | 일지 제출률 (%) |
| 상태 | 활성 / 비활성 |

### 5-2. 알림 이메일 관리

멘토의 알림 이메일은 로그인 이메일과 별도로 설정할 수 있습니다.
알림 이메일이 등록되면 매칭/세션 알림이 해당 주소로 발송됩니다.

**개별 수정:**
1. 알림 이메일 칸 클릭 → 입력 모드 전환
2. 이메일 주소 입력
3. **저장** 클릭

**엑셀 일괄 등록:**
1. 상단 **엑셀 업로드로 일괄 등록** 버튼 클릭
2. 엑셀 파일 선택 (이름 + 이메일 열 포함)
3. 미리보기 확인 (변경 예정 / 동일 / 미등록 / 동명이인 표시)
4. **일괄 반영** 클릭

> 엑셀 파일의 열 이름은 '이름/성명/멘토명'과 '이메일/메일' 키워드를 포함해야 자동 인식됩니다.

### 5-3. 멘토 상태 관리

- **활성/비활성 전환** : 비활성 시 멘티 예약 화면에서 노출되지 않음
- **삭제** : 멘토를 비활성 처리 (소프트 삭제)

---

## 6. 멘티 관리

### 6-1. 멘티 목록 테이블

멘티 이름, 팀, 이메일, 세션 수(예정/완료)를 확인합니다.

### 6-2. 멘티 삭제

**삭제** 클릭 시:
- 진행 중인 세션(대기/확정)이 있으면 경고 표시
- 확인 시 해당 세션 자동 취소 + 슬롯 복원
- 멘티 데이터 삭제 처리

---

## 7. 일지 목록

멘토가 제출한 멘토링 일지를 열람합니다.

### 7-1. 필터

- **멘토** : 특정 멘토의 일지만 조회
- **멘티** : 특정 멘티 관련 일지만 조회
- **기간** : 시작일 ~ 종료일 범위 지정

### 7-2. 일지 카드

각 일지에 표시되는 정보:
- 멘토 → 멘티
- 멘토링 일시
- 유형, 소요 시간, 제출일
- 평가 등급
- 내용 미리보기 (80자)
- **상세 보기** 클릭으로 전체 내용 확인

---

## 8. 요청 관리

멘토/멘티가 보낸 요청에 회신합니다.

### 8-1. 대기 중 요청

- 작성자 이름, 역할, 요청 유형 확인
- 요청 내용 전문 확인
- 회신 입력란에 답변 작성
- **회신 보내기** 클릭 → 상태가 '처리중'으로 변경

### 8-2. 처리 완료 요청

이미 회신한 요청은 하단에 이력으로 표시됩니다.

---

## 9. 공지 작성

### 9-1. 공지 등록

1. **대상** 선택 : 전체 / 멘토만 / 멘티만
2. **제목** 입력
3. **내용** 입력
4. 필요 시 **중요 공지로 등록** 체크
5. **공지 등록** 클릭

> 중요 공지는 목록 상단에 고정 표시되며, 제목 앞에 빨간 느낌표가 붙습니다.

### 9-2. 공지 삭제

각 공지 카드의 **삭제** 버튼으로 제거합니다.

---

## 10. Supabase 연동 (시스템 설정)

### 10-1. 연결 설정

- Supabase URL / Anon Key / Google Client ID 입력
- **저장** 후 **연결 테스트**로 확인

### 10-2. 이메일 큐 관리

- 대기 / 발송중 / 발송완료 / 실패 건수 확인
- **이메일 큐 처리** 버튼으로 대기 중인 메일 수동 발송

---

## 11. 미리보기 기능

관리자는 멘토/멘티 화면을 미리 확인할 수 있습니다.

1. 사이드바 하단의 **멘토 미리보기** 또는 **멘티 미리보기** 클릭
2. 해당 역할의 화면으로 전환 (상단에 미리보기 배너 표시)
3. **관리자로 돌아가기** 클릭으로 복귀

---

## 12. 이메일 알림 발송 현황

관리자 활동에 따라 자동 발송되는 이메일 목록:

| 시점 | 수신자 | 내용 |
|------|--------|------|
| 매칭 제안 시 | 멘토 | 새 매칭 제안 알림 |
| 매칭 확정 시 (멘토 수락) | 멘토 + 멘티 | 매칭 확정 안내 |
| 직접 매칭 확정 시 | 멘토 + 멘티 | 매칭 확정 안내 |
| 멘토가 매칭 거절 시 | 관리자 | 재매칭 필요 알림 |
| 관리자가 세션 취소 시 | 멘토 + 멘티 | 세션 취소 안내 |

---

## 13. 전체 워크플로우 요약

```
[관리자 시점 전체 흐름]

1. 멘토/멘티 등록 (Supabase에서 계정 생성)
2. 멘토 알림 이메일 설정 (멘토 관리 → 개별 또는 엑셀 일괄)
3. 매칭 처리
   - 멘티 신청 기반: 신청 확인 → 멘토 선택 → 제안 → 멘토 응답 대기
   - 직접 매칭: 멘토 + 멘티 선택 → 즉시 확정
4. 멘토링 진행 모니터링 (대시보드 통계)
5. 일지 미제출 관리 (미제출 목록 확인 → 멘토에게 독려)
6. 요청 회신 (멘토/멘티 문의 처리)
7. 공지사항 등록 (전체/멘토/멘티 대상)
```
